# Claude Code

## `[입문]`
* 기본적인 클로드 코드의 사용법
<br><br>

### [CLAUDE.md]
* 프로젝트의 모든 컨텍스트를 담고 있는 핵심 파일
* 주요 포함 내용:
    * 절대 규칙: 프로덕션 DB 쿼리 금지, 환경 변수 커밋 금지
    * 아키텍처: 프로젝트의 전체적인 트리 구조
    * 기술 스택 및 명령어: 사용 언어, 프레임워크, 빌드 및 테스트 명령어
    * 도메인 컨텍스트: 도메인 관련된 지식 주입
    * 코딩 컨벤션: 명명 규칙(PascalCase, camelCase)
* 최적화 전략:
    * 루트의 CLAUDE.md는 300자 이내로 유지하여 토큰 소모를 줄인다.
    * 상세 내용은 폴더별(API, Web 등)로 별도의 CLAUDE.md를 만들어 관리한다.
    * 클로드에게 명령어를 내려 자동으로 내용을 업데이트할 수 있다.
<br><br>

### [트리거 키워드와 커스텀 명령어]
* 특정 키워드 (ex. 배포)를 감지하면 정해진 프로세스를 자동으로 실행할 수 있다.
* 트리거 예시: 테스트 실행 → 실패 시 중단 → 성공 시 깃 애드/커밋/푸시 → 요약 출력
* 슬래시 명령어 형태의 커스텀 명령어를 만들면 탭 자동 완성이 가능하다.
<br><br>

### [주요 단축키 및 제어]
* 터미널 내에서 느낌표로 Bash 명령어 실행이 가능하다.
* Shift + Tab: 플랜(Plan) 모드와 에디트(Edit/Accept) 모드 전환
* 플랜 모드:
    * 클로드에게 파일 수정 권한이 없으므로 먼저 계획을 세우고 사용자 검토를 받는다.
    * 불필요한 코드 생성을 방지하여 토큰을 절약하고 정확도를 높인다.
    * 엔트로픽에서도 플랜 모드로 시작할 것을 권장한다.
    * -> 복잡한 문제 해결할 때는 추천. 간단한건 그냥 해도 됨.
* ESC:
    * ESC 키를 1번 눌러 현재 진행 중인 작업을 즉시 중단(Interrupt)할 수 있다.
    * ESC 키를 2번 눌러 이전 프롬프트 상태로 되돌아가는 리와인드(Rewind)를 할 수 있다.
    * (터미널에서만 가능함. VS code는 클릭해야 가능.)
* 스크린샷 전송:
    * 터미널에 드래그 앤 드랍으로 가능하다.
    * VS code는 더하기 표시를 클릭하면 된다.
<br><br>

### [필수 슬래시 명령어]
* 기본 동작
    * /model: 모델 선택
    * /resume: 이전 대화 복원
    * /config: 기본 세팅
    * /export: 채팅 내보내기
    * /output-style: 기본, 학습 모드, 러닝 모드
* 컨텍스트 관련
    * /context: 현재 토큰 사용량 확인
    * /clear: 컨텍스트 초기화, 새로운 대화로 시작한다.
    * /compact: 컨텍스트 압축, 대화 맥락은 유지하며 토큰만 줄여준다.
* 외부 도구
    * /mcp: supabase, notion 등 도구를 사용할 수 있다.
* 나만의 슬래시 커맨드 만들기도 가능하다.
<br><br>



## `[실전]`
* 컨텍스트 관리와 워크플로우 중심의 활용법
<br><br>

### [세컨드 브레인]
* 세컨트 브레인 구축:
    * 클로드와 작업하며 배운 패턴, 해결책, 의사 결정 근거 등을 마크다운에 기록
    * "방금 진행한 대화에서 코드 패턴, 인사이트, 디버깅 기억해줘." 라고 명령하면 알아서 자동으로 저장된다.
    * 이를 슬래시 명령어에 미리 설정해두면 편하다. 
    * <project>/memory/MEMORY.md 안에 저장된다.
    * 불러올 때는 /memory 를 사용하면 불러올 수 있다.
    * 팀 공유 지식은 CLAUDE.md 에 개인 내용은 MEMORY.md 에 넣는 것 추천.
<br><br>

### [컨텍스트 최적화]
* 모든 정보를 하나의 CLAUDE.md에 몰아넣으면 매 세션마다 불필요한 토큰이 낭비된다.
* 레이지 로딩:
    * 필요할 때만 상세 파일을 읽게 한다.
    * CLAUDE.md에는 핵심 규칙과 참조 정보만 넣는다.
    * 상세 내용은 별도 파일 (API 스펙, DB 스키마 등) 경로로 명시한다.
    * 클로드는 필요할 때만 해당 마크다운 파일을 찾아 읽어 토큰을 절약한다.
* 1 session 1 feature:
    * 컨텍스트 차면 성능이 떨어지는 편이며 `신선한` 세션이 좋다.
    * 한 세션에서는 하나의 기능만 구현하는 습관을 들인다.
    * 작업 끝나면 새 세션에서 진행한다.
* MCP 관리:
    * 사용하지 않는 MCP(노션, 리니얼 등)는 disable 명령으로 비활성화한다.
    * 자주 쓰는 기능만 골라 커스텀 MCP를 만들면 좋다.
<br><br>

### [Mermaid 아키텍처 활용]
* 클로드는 머메이드 다이어그램을 잘 알아듣는다.
* 컨텍스트도 덜 사용하고 이해도 빠른 편이다.
* 미리 자주 쓰는 아키텍처를 머메이드로 정리해두고 레이지 로딩한다. (docs/architecture.md)
<br><br>

### [스크립트 오프로드]
* 무거운 작업은 스크립트 오프로드로 진행한다.
* 컨텍스트를 소비하지 않도록 인간이 대신 실행하고 결과만 준다.
* 스크립트 작성 시키기 -> 인간이 실행 -> 결과만 전달
* 결과에는 전체 내용에 대한 가벼운 요약만 포함하도록 시킨다.
<br><br>

### [실전 개발 프로세스]
* Plan mode 우선 사용
    * 웬만하면 플랜 모드로 진행하는 것을 추천한다.
    * 플래닝 -> 계획서 수정해주기 -> 엑셉트 모드로 실행
* TDD 기반의 스마트 코딩
    * TDD: 테스트 기반 개발
    * 작은 변경 -> 테스트 -> 린트 -> 커밋 -> 반복
* Thinking 로그
    * 닫아놓지 말고 읽기
    * 이상한 방향으로 가면 인터럽트 하고 알려주기
* AI 크로스체크: 대화 복사해서 제미나이 프로에게 검수받기
* AI 신뢰: 에러 발생 시 로그 그대로 넣어주기
* WAT 프레임워크
    * Workflow: 작업 흐름을 plain text로 명확히 정의
    * Agent: 에이전트 팀즈 기능으로 여러 테스크 병렬 처리, 에러 시 Self-healing
    * Tools: 복잡한 스크립트보다 작은 단위의 도구를 조합하여 사용 (Shell 스크립트, MCP 서버 등)

## `[심화]`
* 파워 유저 되기, 에이전틱 엔지니어 되기

### [Skills]
* AI에게 주는 업무 메뉴얼, SKILL.md
* 일정한 품질의 결과물을 얻기 위해 업무 순서, 형식, 확인 사항 등을 정의한다.
* 핵심 기능: 반복 작업 자동화, 일정한 품질 유지, 팀 내 공유
* 비교
    * CLAUDE.md: 코딩 컨벤션, 금지사항, 지침 (범용적인 내용)
    * SKILL.md: 반복 작업에 대한 자동화 레시피 (특정 작업 내용)
* 스킬 로딩
    * 스킬은 디렉토리 단위로 관리되며 SKILL.md가 진입점 역할
    * 1단계(Description): 스킬 이름과 한 줄 설명만 먼저 로드
    * 2단계(Full Prompt): 사용 시점에만 전체 내용을 온디맨드 로드
    * 스킬은 설명만 넣으므로 매우 효율적이다.
    * MCP는 스킬이랑 비슷한 역할이지만, 모든 도구 설명이 매 턴에 들어가서 비효율적이다.
* 적합한 사용처
    * 시스템 프롬프트: 핵심 규칙 / 매번 입력되고 결과가 불안정하다.
    * MCP: 외부 서비스 연동 가능 / 연결된 모든 도구 설명이 로드되어 무겁다.
    * 서브 에이전트: 대량 탐색 / 정의가 은근 길어서 무겁다.
    * 스킬: 반복 작업 / 별도 컨텍스트를 하용하지는 않는다.
    * 서브 에이전트와 스킬을 잘 섞어쓰는게 중요하다.
* Description
    * 스킬 품질에 가장 중요한 항목이다.
    * 핵심 기능, 트리거 표현 3개 이상, 차별화를 포함한다.
    * (ex. README.md 생성 스킬. '리드미로 정리해줘', '프로젝트 내용 정리해줘' 요청 시 트리거)
* 내장 기능으로 skill-creator plugin이 존재한다. (스킬 만들어줘.)
* PPT 생성 같은 비개발 관련 스킬도 잘 하는 편이다.
<br><br>

### [Sub Agent]
* 메인 클로드 안에서 별도의 작업 공간을 가진 도우미를 생성한다.
* 핵심 기능: 병렬 처리, 컨텍스트 보호와 절약, 전문화, 재사용
* 내장 에이전트
    * Explorer: 코드 탐색 및 구조 파악 특화
    * Plan: 계획 수립
    * General-purpose: 복잡한 연구 및 수정
    * Bash: 터미널 명령 실행
    * Guide: 클로드 기능 안내
* 서브 에이전트 만들기:
    * /agent 명령어를 통해 생성
    * 모델 선택, 색깔 선택, 권한 설정, 메모리 활성화 등이 가능하다.
    * 에이전트 파일은 프로젝트 폴더나 개인 폴더에 저장하여 사용 범위를 결정할 수 있다.
    * 격리 패턴 (독립 사용), 병렬 패턴 (동시 사용), 체인 패턴 (순차 사용) 등이 있다.
* 에이전트 Teams:
    * 여러 클로드 코드 인스턴스가 팀으로 협업하는 기능이다.
<br><br>

### [Hooks]
* 내부 자동화 엔진으로 특정 이벤트 발생 시 액션 (린트, 포맷, 노티 등)을 실행할 수 있다.
* 훅 만들기:
    * 클로드에게 명령하거나 setting.json 수정하여 가능
    * 훅이 안끝나면 클로드가 멈추기 때문에 가벼운 작업만 훅으로 만든다.
* 사용 가능한 주요 이벤트:
    * Pre-use: 도구 실행 전, 입력값 검증이나 승인 및 차단
    * Post-use: 도구 실행 후, 린트(Lint)나 포맷 적용 등 후처리
    * Notification: 응답 대기, 맥 OS 알림이나 효과음
    * Stop: 턴 종료, 최종 보고서 생성이나 상태 저장
<br><br>

### [심화 팁]
* 멀티 인스턴스:
    * 여러 터미널 탭에서 다른 클로드 인스턴스로 병렬 개발할 수 있다.
    * iTerms 활용을 추천한다.
* 깃 워크트리:
    * 하나의 저장소에서 브랜치별로 별도의 폴더를 생성한다.
    * 체크아웃 없이 여러 브랜치를 동시에 작업할 수 있게 해준다.
    * 명령어 claude -w <워크트리 이름> 을 하거나 CLAUDE.md에 지침을 적으면 된다.
