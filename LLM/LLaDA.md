## [`논문 요약`]

### [저자의 의도]
* 기존 autoregressive 방식이 아닌 diffusion 방식을 사용해보자.
    * autoregressive: 다음 단어 예측, ARM
    * diffusion, 전체를 한번에 점진적으로 예측, LLaDA
* 핵심 LLM 역량들이 ARM 덕분이라는 보편적인 가설에 의문점을 제기한다.
* ARM 방법론 없이 LLM의 capability 능력이 발현되는지 조사해보자.
<br><br>

### [기존 문제점]
* ARM
    * LLM을 구현하는 핵심 이론은 2가지로 나눌 수 있다.
    * 생성 모델링 이론: 인간의 언어의 실제 확률 분포와 모델 분포 사이의 차이를 최소화하여 언어의 패턴을 학습
    * autoregressive 이론: 다음 토큰 예측 패러다임으로 불리는 autoregressive 모델링(ARM)
    * 여기서 두번째에 해당하는 autogressive. 이게 정말 유일한 방법일까.
    * 특히 scalability (확장성) 측면을 보면 ARM 보다는 생성 모델링 이론이 중요하다.
    * 생성 모델링 이론에서 파생된 트랜스포머 아키텍처, 모델 크기, 데이터 크기 등이 이 scalability를 가져왔다.
    * Vision에서는 디퓨전 트랜스포머가 큰 성공을 이뤘으므로 위와 같은 생각을 뒷받침 해준다.
    * 특히 instruction-following, in-context learning 두 task는 ARM 의 독자적 특징이 아니다.
    * 생성 모델링으로 만든 모델은 모두 보이는 특징이라는 것이다.
    * ARM은 일종의 lossless data compressor (손실 없는 데이터 압축기)로 해석할 수 있다.
    * 저자들은 충분한 표현력을 갖춘 확률모델이 있다면 언제든 ARM을 대체할 수 있다고 주장한다.
    * ARM에는 몇가지 단점들이 존재한다.
    * 왼쪽에서 오른쪽으로만 생성이 가능하기 때문에 반대로 진행하는 "reversal reasoning"이 불가능하다.
    * 또한 학습에서 보지못한 새로운 데이터에 대한 일반화 성능 "generalization capability"가 매우 떨어지는 편이다.

### [해결 아이디어]
* LLaDA Overview
    * Large Language Diffusion with mAsking
    * ARM 방법론과 반대되는 masked diffusion model (MDM)을 사용한다.
    * MDM: 데이터를 점진적으로 마스킹하여 정보를 제거하고, 다시 이를 복원하는 과정을 통해 데이터의 분포를 학습
    * "전방향 데이터 마스킹 과정"과 이를 복원하는 "역방향 근사 과정"을 위해 mask predictor를 학습한다.
        * Forward data masking process: 입력 텍스트의 토큰들을 무작위로 가리는 과정
        * Approximate its reverse process: 마스킹된 상태에서 시작하여 원래 데이터로 단계적으로 복원
    * 이러한 디자인으로 LLaDA는 단방향이 아니라 양방향으로 예측할 수 있는 모델 확률분포를 구축한다.
    * 따라서 reversal reasoning이 가능하게 되는 것이다.
    * 저자들은 학습 데이터 전처리, pre-training, fine-tuning, 평가 파이프라인을 모두 최적화했다.
    * Fig 2
    * Pre-training: (시스템 프롬프트, 응답) 모든 토큰에 대하여 독립적으로 랜덤 마스킹된 텍스트에서 학습
    * Fine-tuning: 응답 토큰에 대하여 랜덤 마스킹된 텍스트에서 학습
    * Sampling: t=1 (전체 마스킹) 에서 t=0 (마스킹 없음)으로 가는 디퓨전 시뮬레이션
    * (flexible remask 전략 사용, 각각의 스텝에서 모든 마스크를 동시 예측)
    * flexible remask: 모든 마스크 동시 예측, confidence가 낮은 부분은 다시 마스킹, 다시 동시 예측
* Probabilistic Formulation
    * ARM과 다르게 forward process와 reverse process로 정의할 수 있다.
    * t가 0에서 1 사이의 실수일 때 0은 마스킹이 없는 상태, 1은 전체 마스킹 상태로 가정하자.
    * forward는 전체 컨텍스트를 점진적으로 마스킹을 진행하여 t=1에서 완전히 마스킹한다.
    * (토큰에 관점에서 독립적. 토큰은 마스킹을 하고 안하고만 존재. 노이즈가 절반만 먹은 토큰같은 개념은 없음.)
    * (전체 컨텍스트 관점에서 점진적. 전체 중에 절반 정도의 토큰이 가려졌다면 노이즈가 절반만 먹은 것.)
    * 개인적인 의견: 필자는 Vision 모델 전문가로 여기서 뭔가 이상한 점을 느꼈다. 이거 뭔가 DiT 방식과 MAE 방식을 짬뽕한 신기한 방법론인데? 라는 생각이다. 토큰을 통째로 가리는건 사실 디퓨전 보다는 MAE에 가깝다. DiT 처럼 하려면 노이즈를 점진적으로 먹여야하는데 그런 방식이 아니다. DiT 백본을 쓰는 모델은 MDLM, Mercury 등이 있다. 또한 LLaDA 방식과 MDLM 방식의 장점만 취하는 모델 (ex. BD3LM, 제미나이 디퓨전)도 존재하고, 완전히 새로운 이론인 에너지 함수를 가져와서 서커스하는 모델 (ex. EDLM)도 있다.
    * LLaDA의 핵심은 mask predictor 이다.
    * 토큰이 마스킹된 시퀀스 x_t를 인풋으로 받아 마스킹된 모든 토큰을 동시에 예측한다.
    * 학습에 사용하는 loss function은 크로스 엔트로피 함수로 Eq 3과 같이 생겼다.
    * 잘 보면 마스킹된 토큰에 대해서만 loss를 계산하는 트릭이 적용되어 있다.
    * BERT에서는 마스킹을 고정된 비율로 하는데 LLaDA는 1~0 사이의 무작위 값을 사용한다.
    * 이는 sacling 관점에서 매우 유리한데, 같은 데이터에서도 더 다양하고 많은 학습 데이터를 뽑아낼 수 있다.
    * (+대충 본인들이 만든 loss function에 대한 통계적 정당성 해명하는 문장들)
* 출력 길이 제어 매커니즘
    * SFT 설명에서 디퓨전 모델에서 매우 중요한 `출력 길이 제어 매커니즘` 관련 언급을 먼저 설명하겠다.
    * 저자들은 각 미니 배치 내의 모든 데이터 길이를 동일하게 맞추기 위해서 트릭을 사용한다.
    * 여기서 |EOS|를 활용하여 무조건 최대 시퀀스 길이인 4096 토큰에 딱 맞도록 만든다.
    * 예시1: 질문+답변이 4096 토큰이 안됨, 4000 토큰임 -> 뒤에 96개 |EOS| 토큰을 붙인다.
    * 예시2: 질문+답변이 딱 4096 토큰임, 혹은 4096의 배수임 -> 그대로 사용한다.
    * 예시3: 질문+답변이 4096보다 애매하게 김, 5000 토큰임 -> 일단 4096은 쓰고, 904 뒤에 3192개 |EOS| 토큰을 붙인다.
    * 학습 중에는 |EOS| 토큰을 일반 토큰과 동일하게 취급하여 학습을 진행한다.
    * 추론 시에는 |EOS| 토큰을 제거하여 LLaDA가 응답 길이를 자동으로 제어할 수 있도록 해준다.
    * (어쩌면 그래 보이는 것 처럼 트릭을 준다.)
* Pre-training
    * 기존 LLM과 유사하게 트랜스포머를 mask predictor로 사용한다.
    * 그런데 LLaDA는 causal mask를 사용하지 않고 인풋 전체를 다 본다.
    * 여기서 causal mask란 LLM 학습에 사용하는 미래 토큰을 다 가리는 방법을 말한다.
    * LLaDA 연구자들은 1B와 8B 2개의 모델을 학습했다.
    * Table 5, 6
    * 기본적으로 LLaMA 3의 아키텍처를 가져와 일부를 변형해 학습했다.
    * RMS Norm 사용 안정화, ReLU 대신 SwiGLU 사용, 기본 PE 대신 RoPE 사용
    * AdamW 옵티마이저, weight decay 0.1, Warmup-Stable-Decay LR 스케쥴러 사용
    * lr = 4 x 10^-04, batch_size = 1280
    * 바닐라 멀티헤드 셀프 어텐션 레이어 대신에 grouped query attention를 사용했다.
* Supervised Fine-Tuning
    * LLaDA의 capability를 올리기 위해 SFT도 진행했다.
    * SFT는 기존 autoregressive 모델들이 하는 것과 동일하게 진행했다.
    * SFT의 경우 3 에포크 동안 학습한다. (일반 LLM들은 1 에포크만 학습하는 편)
    * Warmup-Stable-Decay LR 스케쥴러 사용
    * lr = 2.5 x 10^-05, batch_size =256
* Inference
    * LLaDA는 추론에 앞서 지정한 생성 길이만큼 응답 영역을 전부 마스크로 채우는 초기화 작업을 한다.
    * 완전히 마스킹된 상태가 있어야 LLaDA가 추론을 시작할 수 있기 때문이다.
    * 예측은 항상 마스킹된 모든 토큰에 대하여 진행한다.
    * 그러니까 일단은 모든 영역을 다 예측한 뒤에 `다시 일부분을 마스킹, remasking` 한다.
    * 이 마스킹할 일부분은 예측한 토큰들 중에서 가장 컨피던스가 낮은 불확실한 토큰들이다.
    * 그리고 마스킹된 모든 영역을 다시 다 예측하고 또 리마스킹하는 작업을 반복한다.
    * 이 예측 반복 횟수 t는 논문에서 `샘플링 횟수`라고 표현하며, 하이퍼파라미터 값으로 조절 가능하다.
    * 생성할 길이가 얼마나 긴지에 대해서도 하이퍼파라미터 값으로 두고 조절 가능하다.
    * 저자들은 이 방식을 `diffusion sampling` 이라고 부른다.
    * 생성이 끝나면 |EOS| 토큰 이후의 모든 토큰은 폐기처리 한다.
    * 학습 데이터 자체가 가변적인 길이를 가졌기 때문에 생각보다 이 생성길이에 민감하지 않다고 주장한다.
    * (진짜로? 신기한데? 과연 진짜 그럴까?)
    * 원칙적으로는 이 리마스킹을 낮은 컨피던스 토큰이 아니라 무작위로 해야한다고 한다.
    * 하지만 LLM에서도 샘플링할 때 annealing tricks을 쓰기 때문에 여기서도 사용했다고 설명한다.
    * (여기서 어닐링 트릭이라는 것은 LLM 하이퍼 파라미터의 temperature를 말한다.)
    * LLaDA는 특이하게도 샘플링에 유연성이 있어서 3가지 추론 방식이 모두 동작한다.
    * (기존 autoregressive 추론, block diffusion 추론, diffusion sampling 추론)
    * 그런데 실험을 해본 결과 diffusion sampling이 가장 좋은 성능을 보였다.
    * (필자의 생각: 당연한 결과. 디퓨전으로 학습했으니 디퓨전이 잘될거고, 샘플링은 성능을 높이는 트릭이다.)

### [결과 분석]
* Scalability
    * 아키텍처, 학습 데이터 등이 완전히 동일한 ARM과 LLaDA 1B 모델을 학습하여 평가했다.
    * 조금 더 큰 모델들은 ARM과 LLaDA에 약간의 차이가 존재하고 완전히 동일하지는 않다.
    * Table 5
    * Table 5에 자세하게 적혀있다.
    * Fig 3
    * 이 그림에서 보이는 FLOPs는 학습량 (데이터 노출량)을 의미한다.
    * 따라서 학습에 따라서 모델이 어떻게 성장하는지를 보여주는 그래프이다.
    * FLOPs는 6ND로 계산되며, N은 임베딩 제외 파라미터수, D는 학습 데이터 토큰 수 이다.
    * 6개의 task에서 학습이 잘 이루어질 뿐만 아니라, 기존 ARM 아키텍처보다 scalability이 좋다고 주장한다.
    * 아마도 MMLU에서 최고점이 더 높고 GSM8K에서 기울기가 가파르기 때문으로 보인다.
    * 저자들은 이것이 아키텍처 차이에서 비롯된 것이라고 가설을 세웠다.
    * (필자의 생각: 주장이 맞는지 잘 모르겠음. 필자는 사전학습을 2년동안 진행한 연구원임. 필자가 볼때는 오히려 학습의 불안정성으로 붕괴 가능성이 높아 보임. 평탄하고 좋은 loss landscape를 형성한 것으로 보이지는 않음. (MMLU, PIQA) 그리고 기울기가 높다고 scalability가 좋은건 아님. 오히려 갑자기 세츄레이션 되버리는 경향이 생기고 무난하게 상승하는게 끝까지 학습해보면 더 나은 경우도 많음. (GSM8K, HumanEval))
    * 대표적인 디퓨전 모델인 MDM 저자들은 ARM에 비해 MDM이 16배 많은 학습량이 필요하다고 말한다.
    * 자신들이 스스로 "디퓨전 모델은 효율성이 떨어진다" 라고 말한 것이다.
    * 하지만 Fig 3을 보면 확실하게 이 문제점은 해결한 것으로 보인다.

    * 평가 데이터셋의 종류는 다음과 같다. (일반 질의, 수학 과학, 코딩, 중국어)
